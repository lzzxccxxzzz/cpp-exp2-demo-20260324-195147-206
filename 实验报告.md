# 实验二 栈的应用实验报告

## 一、实验名称

实验二 栈的应用

## 二、实验目的

1. 掌握顺序栈的存储结构和基本操作。
2. 理解栈“后进先出”特性在实际问题中的应用。
3. 学会使用顺序栈实现十进制数向其他进制的转换。
4. 学会使用顺序栈判断表达式中括号是否匹配。

## 三、实验内容

1. 使用顺序栈，实现任意十进制数的 `m` 进制输出，其中 `0 < m < 10`。
2. 使用顺序栈，实现判断表达式括号是否匹配的算法，括号类型限制为两种：`()` 和 `[]`。

## 四、实验环境

- 操作系统：Windows
- 编程语言：C++
- 源文件：`main.cpp`

## 五、需求分析

### 1. 十进制数转换为 m 进制

十进制数转换为其他进制时，通常采用“除基取余，逆序输出”的方法。  
因为每次得到的余数顺序与最终结果相反，所以非常适合使用栈来暂存余数，再按出栈顺序拼接成最终字符串。

处理过程如下：

1. 将十进制整数 `n` 不断除以 `m`。
2. 每次把余数压入顺序栈。
3. 当商为 0 时停止。
4. 将栈中元素依次弹出，得到目标进制结果。

说明：

- 程序支持 `m=1~9`。
- 当 `m=1` 时，程序按一进制形式输出若干个字符 `1`。
- 当输入 `n=0` 时，直接输出 `0`。
- 程序对负数也进行了兼容处理，结果以前导负号表示。

### 2. 括号匹配判断

括号匹配问题同样适合使用栈处理。扫描表达式时：

1. 遇到左括号则压栈。
2. 遇到右括号时，判断栈是否为空。
3. 若栈空，说明右括号多余，匹配失败。
4. 若栈非空，则弹出栈顶左括号，判断类型是否匹配。
5. 扫描结束后，若栈仍非空，说明存在未匹配的左括号。
6. 若整个扫描过程都没有出错且最终栈为空，则括号匹配成功。

## 六、数据结构设计

本实验定义了一个模板顺序栈 `SeqStack<T>`，采用动态数组作为底层存储空间，主要成员包括：

- `data_`：存储栈元素的动态数组。
- `capacity_`：当前数组容量。
- `top_`：栈顶下标，空栈时为 `-1`。

主要操作如下：

- `push(const T& value)`：入栈。
- `pop()`：出栈并返回栈顶元素。
- `top()`：读取栈顶元素。
- `empty()`：判断是否为空栈。
- `size()`：返回栈中元素数量。

当数组空间已满时，程序会自动扩容，以保证表达式较长时仍能正常运行。

## 七、算法设计

### 1. 十进制转 m 进制算法

#### 算法思想

利用“除基取余”方法生成每一位数字，再借助顺序栈逆序输出。

#### 算法步骤

1. 输入十进制整数 `n` 和目标进制 `m`。
2. 若 `n=0`，直接返回字符串 `"0"`。
3. 若 `m=1`，输出 `|n|` 个字符 `1`，若 `n<0` 则加负号。
4. 对 `|n|` 重复执行：
   - 求余数 `n % m`；
   - 将余数对应字符压栈；
   - 更新 `n = n / m`。
5. 将栈中元素依次出栈并连接成字符串。
6. 若原数为负，结果前加 `-`。

#### 时间复杂度

- 时间复杂度：`O(log_m n)`。
- 空间复杂度：`O(log_m n)`。

### 2. 括号匹配算法

#### 算法思想

遇左括号入栈，遇右括号与栈顶左括号进行匹配。

#### 算法步骤

1. 从左到右扫描表达式中的每个字符。
2. 若为左括号 `(` 或 `[`，则压栈。
3. 若为右括号 `)` 或 `]`：
   - 若栈空，则匹配失败；
   - 否则弹出栈顶元素并检查括号类型是否对应。
4. 扫描结束后：
   - 若栈为空，则匹配成功；
   - 若栈非空，则说明存在未闭合的左括号，匹配失败。

#### 时间复杂度

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(n)`。

程序采用菜单方式组织：

## 十一、实验总结

通过本次实验，我进一步掌握了顺序栈的定义和基本操作，理解了栈结构“后进先出”的核心特点。在十进制转 `m` 进制问题中，利用栈可以方便地将逆序产生的余数转换成正确顺序的结果；在括号匹配问题中，利用栈可以高效地保存尚未匹配的左括号，并在遇到右括号时完成配对判断。

本实验表明，栈是一种非常适合处理“逆序输出”和“成对匹配”问题的数据结构。通过编写和调试程序，我对顺序栈在实际问题中的应用有了更加直观的认识，也提高了使用 C++ 进行抽象数据结构实现的能力。
